Contraintes¶

Plate-forme technique

Exemple de ce qui a été livré à Airbus pour les spec hard et soft.

Le périmètre concerné :

  • brique de gestion de configuration : Puppet
  • brique de gestion du cycle de vie des machines : Foreman

Besoins en terme de ressources matérielles : - 2 vcpu - 4 Go RAM - 50Go disque dur - 1 interface réseau

Nous apportons une VM.

La VM peut être déployée sur deux hyperviseurs différents : KVM ou VMware vSphere Cluster (pas un ESXi en standalone).

La VM usine sera une Debian 8 Jessie sour format OVA pour VMware, en format KVM dans le cas inverse

Il y aura des clefs ssh à échanger avec cet hyperviseur pour que l’usine puisse le piloter.

En termes de ports ouverts : - La VM usine doit accéder au port 22 de l’hyperviseur - La VM usine accède aux ports TCP 5901 à 5964 de l’hyperviseur - Le poste utilisateur accède aux ports TCP 69,80, 443, 5910-5930, 8140, 8443, 22 de l’usine - La VM usine doit accéder à Internet

Afin d’automatiser au maximum le déploiement des services, nous avons besoin d’un serveur qui fait DNS+DHCP. L’usine est compatible avec les services suivants : - Linux Bind - Microsoft DNS Server - Linux ISC DHCP - Microsoft DHCP Servers
