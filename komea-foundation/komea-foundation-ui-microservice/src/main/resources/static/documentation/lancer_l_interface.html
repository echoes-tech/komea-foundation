Lancer l’interface¶

Spécification générale¶

Il s’agit d’afficher l’écran d’accueil de KSF.

(P2) l’utilisateur pourra préférer se remettre dans le dernier contexte d’utilisation :

  • se reconnecter
  • rouvrir le dernier projet utilisé
  • ...

Spécification détaillée¶

L’accès à KSF se fait via une interface web.

Dans un navigateur, on saisit une URL pointant sur KSF. Exemple : http://ksf.metrixware.com

La configuration de l’usine peut être - globale (fichier de conf / BDD ?) - spécifique à chaque projet (choix parmi les outils disponibles dans la conf globale)

Le bandeau en haut contient :

  • à gauche, le nom de la config KSF active (pour switcher de config, on relance l’interface et basta)
  • un message de bienvenue
  • à droite, les boutons “login”, “help” et “preferences”

Lire le fichier de config KSF et construire une page html : Pour chaque outil, inclure sa vue de présentation (pourrait faire partie des paramètres de l’outil). On définit au niveau général, un css pour afficher les outils en format graphique et (P2) un css pour les afficher sous forme de liste. Cet affichage servira pour la création de Project (on met en display:none la checkbox de sélection par exemple).

Pour répondre à ces besoins, les différents outils devront être ajouté à KSF sous forme de plugins. Ces plugins doivent avoir une interface capable de fournir les données suivantes :

  • Identifiant
  • Nom
  • Description
  • Propriétés (affichage ou non par exemple, dans le cas de Git il n’y a rien à afficher puisque sa gestion se fera avec un autre outil)
  • Résumé (explication de l’utilité de l’outil en question)
  • Affichage (ce qu’on doit afficher)

Le plugin devra également pouvoir faire les actions suivantes :

  • Créer un projet
  • Créer une fonctionnalité (au sens gitflow)
  • Fermer une fonctionnalité
  • Créer une Release
  • Publier une Release
  • Créer un hotfix
  • Fermer un hotfix
  • Générer des environnements
  • Détruire le Projet
  • Dupliquer le Projet
