Connecteur Foreman¶

Définitions :¶

  • Environnement (au sens Foreman) = ensemble de modules puppets dans une version donnée installables sur une ou plusieurs machines. Cet environnement est listé dans Foreman.
  • Surcharge de paramètres de modules : ensemble de variables/valeurs à appliquer aux modules.
  • Cible = environnement + OS + profil d’ordinateur + surcharge

Création d’une Cible (Target)¶

Le widget permet de créer une Cible. Une Cible est une configuration qui pourra être utilisée par la suite pour créer très rapidement de nouvelles VMs.

Pour créer une cible, il faut spécifier :

  • Un nom : C’est le nom qui permettra d’identifier la Cible.
  • Un Environnement : C’est l’ensemble de modules Puppet qui pourront être appliqués à la Cible.
  • Un système d’exploitation : C’est le système d’exploitation qui sera installé sur la future VM.
  • Un profil d’ordinateur : C’est le type de VM qui sera créée.
  • Un champs texte : Il permet de régler les différents paramètres correspondant aux modules Puppet embarqués dans l’Environnement. Ce champs est pré-rempli avec les valeurs par défaut pour ces modules.

Création d’un environnement (Environment)¶

Le widget permet de créer un Environnement. Cet Environnement est une liste de modules Puppet avec leur version. Il peut être utilisé pour choisir la liste des modules Puppet disponibles pour la création d’une Cible.

Pour créer un Environnement, il est possible de choisir un Environnement pré-existant (Source Environment). C’est plus rapide que de créer un nouvel Environnement de zéro.

Pour créer un Environnement, il faut spécifier :

  • Un nom : C’est le nom qui permettra d’identifier l’Environnement.
  • Un champ texte : Il permet de décrire l’Environnement souhaité. Un example est pré-rempli dans le champs. Il est possible de choisir parmi les Environnements existants afin de pré-remplir le champs.
