Architecture fonctionnelle¶

Schéma fonctionnel de l’usine logicielle

La fonction “piloter” couvre 3 fonctions :

  1. suivre = aggrégation de données dans un tableau de bord = KOMEA-Dashboard
  2. orchestrer = commander de manière synchronisée les outils de l’usine = KOMEA-CC
  3. superviser = surveiller le bon fonctionnement de l’usine logicielle = Push Business

La fonction “Valider les développements” recouvre :

  1. l’Analyse Qualité du code
  2. les revues peer to peer (fonction manuelle par excellence)
  3. les test unitaires (faudrait-il les modéliser à part?)

La fonction “Gestion de Projet” recouvre:

  1. gestion des tickets clients
  2. gestion des bugs
  3. gestion des taches
  4. gestion des ressources
  5. planning
  6. .../...

On ajoute la fonction “Gestion des Exigences et des Tests” qui est une demande récurente actuellement.
